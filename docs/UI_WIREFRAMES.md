# UI Wireframes & Screen Descriptions

What the player sees on screen during each phase of gameplay. Describes layout, elements, and interaction patterns for every view.

## Covering:

- **System Map / Scanner View** — Star system overview with celestial bodies, wreck contacts as blips on a radar/oscilloscope display. How contacts are revealed (Discovery Sweep), how focus targeting works, how Deep Scan is initiated. Element placement: radar sweeps, contact list, scan controls.

- **Wreck Cross-Section (Processing View)** — The primary interaction screen. Wreck shown as a wireframe/radar cross-section on a CRT. Sections highlighted as discrete targets with integrity bars and resource labels. Extractor deployment UI. Active extractor indicators. Stress meter (visual gauge). Probes shown as small icons flying routes.

- **Ship Systems Panel** — Six system bars (Hull, Life Support, Propulsion, Maw Core, Scanner Array, Power Grid) with current % and status text. Resource inventory display. Repair allocation interface — drag resources to systems. Integrity readout (overall ship health).

- **Data Spool Player** — A CRT terminal view. Spool selection interface. Playback controls: play, pause, rewind, speed. Text scroll animations. Static/noise effects for corrupted data. Audio playback indicator.

- **Upgrade Shop** — Three-column layout (Ship Systems / Scanner Array / Cognitive Evolution). Upgrade cards with icon, name, current level, next level cost, prerequisites. Purchase button. Overclock slot.

- **Region Progression Map** — Abstract map showing regions from outer arm to galactic core. Current location, accessible regions, locked regions with unlock conditions. Threat indicators.

- **HUD / Persistent Elements** — Always-visible elements: Integrity bar, resource counter, active probe count, current system name. Scan button (when in system view). Menu access.

- **CRT Aesthetic Notes** — Scanlines, phosphor afterglow, monochrome palettes (amber default, green for data terminals, red for emergencies). Screen curvature. Button/switch click targets styled as physical controls.
